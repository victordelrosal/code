What is GitHub & Why Do I Need It?

Understanding the basics before we start

Your Code Needs a Home on the Internet

Right now, the webpage you created in the Hello World guide only exists on your computer. If you want to share it with friends, family, or the world — you need to put it online.

GitHub is a website where developers store their code. Think of it like Google Drive or Dropbox, but specifically designed for code. And here's the best part: GitHub can also host your website for free!

What GitHub Does

  • Stores your code online (backup!)
  • Hosts websites for free
  • Tracks changes to your files
  • Lets others see your work
  • Used by every professional developer

What You'll Get

  • A free GitHub account
  • Your code stored safely online
  • A real website URL to share
  • A professional portfolio start
  • Skills every coder needs

Step 1: Set Up Your Project Folders

Organize your code like a professional

Why Organization Matters

Before we go any further, let's set up a proper folder structure. As you build more projects, you'll thank yourself for staying organized from day one. Professional developers keep all their code in one place.

Required Setup

Please follow these steps exactly. The folder structure matters for everything we do going forward.

Create Your Projects Folder

Mac Instructions

  1. Open Finder (the blue smiley face icon in your dock)
  2. In the left sidebar, click on your username (it has a house icon)
  3. Right-click in the empty space and select "New Folder"
  4. Name it exactly: projects (all lowercase)
  5. Press Enter to confirm

Windows Instructions

  1. Open File Explorer (the folder icon in your taskbar)
  2. In the left sidebar, click on "This PC"
  3. Double-click on your C: drive or Documents
  4. Right-click in the empty space and select "New" ‚Üí "Folder"
  5. Name it exactly: projects (all lowercase)
  6. Press Enter to confirm

Create Your Hello World Project Folder

  1. Open your new projects folder
  2. Create another new folder inside it
  3. Name it exactly: helloworld (all lowercase, no spaces)

Critical: Rename Your File!

If you saved your file as helloworld.html in the previous guide, you need to rename it to index.html and move it into your new helloworld folder.

Why? When someone visits your website, the server automatically looks for a file called index.html. It's like the "front door" of every website.

Move and Rename Your File

  1. Find your helloworld.html file (probably on your Desktop)
  2. Drag it into your projects/helloworld folder
  3. Right-click the file and select "Rename"
  4. Change the name from helloworld.html to index.html
  5. Press Enter to confirm

Checkpoint

You should now have: projects/helloworld/index.html

Double-click index.html to make sure it still opens in your browser and shows your Hello World page.

Step 2: Create Your GitHub Account

Sign up for free in 2 minutes

Sign Up for GitHub

  1. Go to github.com
  2. Click the "Sign up" button (top right)
  3. Enter your email address
  4. Create a password
  5. Choose a username

Choosing a Username

Your username becomes part of your website URL (username.github.io), so choose wisely!

  • Good: johndoe, jane-smith, jsmith2026
  • Avoid: xXx_gamer_xXx, temp123456
  1. Complete the verification puzzle (proves you're human)
  2. Check your email and click the verification link
  3. Answer the personalization questions (or skip them)
  4. Select the Free plan

Account Created!

You now have a GitHub account. Welcome to the world of coding!

Step 3: Create a Repository

Make a home for your project on GitHub

What's a Repository?

A repository (or "repo") is like a folder for your project on GitHub. Each project you create gets its own repository.

Create Your First Repository

  1. Make sure you're logged into github.com
  2. Click the + icon in the top right corner
  3. Select "New repository"

Fill In the Repository Details

Repository Settings

Repository name: helloworld
Description: My first website (optional)
Public/Private: Select Public
Add a README: Leave unchecked
  1. Click the green "Create repository" button

You'll see a page with setup instructions. Don't worry about these — we'll use an easier method to upload your files.

Repository Created!

You now have an empty repository waiting for your code.

Step 4: Upload Your Website Files

Get your code onto GitHub

Upload Your index.html File

  1. On your repository page, look for the link that says "uploading an existing file" (in the quick setup section)
  2. Click it

If you don't see that link, click the "Add file" button and select "Upload files".

  1. Open your projects/helloworld folder on your computer
  2. Drag your index.html file into the browser window where it says "Drag files here"
  3. Wait for the file to upload (you'll see a green checkmark)

Commit Your Changes

What's a Commit?

A commit is like saving a snapshot of your project. Every time you make changes, you "commit" them with a short description of what you did.

  1. In the "Commit changes" section at the bottom, you'll see a text box
  2. Type a short message like: Add my first webpage
  3. Click the green "Commit changes" button

File Uploaded!

Your code is now on GitHub! You should see index.html listed in your repository.

Step 5: Turn On GitHub Pages

Make your website live on the internet

What's GitHub Pages?

GitHub Pages is a free service that turns your repository into a live website. No servers to set up, no hosting fees — just flip a switch!

Enable GitHub Pages

  1. On your repository page, click "Settings" (the gear icon near the top)
  2. In the left sidebar, scroll down and click "Pages"
  3. Under "Source", you'll see a dropdown that says "None"
  4. Click it and select "main" (or "master")
  5. Leave the folder as "/ (root)"
  6. Click "Save"

Wait for Deployment

GitHub will now build your website. This usually takes 1-2 minutes.

  1. Refresh the page after a minute
  2. You should see a message: "Your site is live at..." with a URL

Your Website is Live!

Click the link to see your website on the real internet!

Your URL will be: https://yourusername.github.io/helloworld

Don't See It Yet?

If you see a 404 error, wait a few more minutes and try again. GitHub Pages can take up to 10 minutes to deploy the first time.

Making Changes to Your Website

How to update your site after it's live

Edit Your Code on GitHub

You can edit your website directly on GitHub without downloading anything!

  1. Go to your repository on GitHub
  2. Click on index.html
  3. Click the pencil icon (top right of the file view) to edit
  4. Make your changes
  5. Scroll down and click "Commit changes"
  6. Wait 1-2 minutes for the site to update
  7. Refresh your website to see the changes!

Or Edit Locally and Re-Upload

  1. Edit index.html on your computer using Sublime Text
  2. Go to your repository on GitHub
  3. Click "Add file" ‚Üí "Upload files"
  4. Drag your updated index.html file in
  5. Commit the changes

Pro Tip

For simple changes, edit directly on GitHub. For bigger updates, edit locally and upload. As you advance, you'll learn to use Git (the command-line tool) for a smoother workflow.

Troubleshooting

Common problems and solutions

Pro Tip: Use AI for Help

Stuck? Describe your problem to your LLM of choice (ChatGPT, Claude, Gemini). You can even share screenshots — AI can read images!

Common Issues

"404 - Page Not Found"

  • Wait a few more minutes — GitHub Pages can take up to 10 minutes
  • Make sure your file is named exactly index.html (not Index.html or index.HTML)
  • Check that GitHub Pages is enabled in Settings ‚Üí Pages
  • Make sure the source is set to "main" branch

"I can't find the Settings or Pages option"

Make sure you're on your repository page (not your profile page). The URL should look like: github.com/yourusername/helloworld

"My changes aren't showing up"

  • Wait 1-2 minutes after committing
  • Hard refresh your browser: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
  • Try opening in an incognito/private window

"The page looks different than on my computer"

Make sure you uploaded the most recent version of your index.html file. The file on GitHub might be an older version.
