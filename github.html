Quick Start: Your First GitHub Repository

5 steps to publish your first website

Step 1 — Create a New Repository

  1. Go to github.com/new
  2. Type firstrepo in the Repository name field
  3. Scroll down and click Create repository

Step 2 — Create Your HTML File

In the blue box area, find the text:

"Quick setup, if you've done this kind of thing before"

  1. Below that, click creating a new file
  2. Name the file index.html and paste in this code:
<!DOCTYPE html>
<html>
<head>
    <title>First repository</title>
</head>
<body>
    <h1>Business Intelligence test</h1>
</body>
</html>
  1. Click the green Commit changes... button
  2. If there's no message pre-filled, type initial commit, then click Commit changes again

Step 3 — Publish Your Page

  1. Click the Settings tab
  2. In the left-hand menu, scroll down and select Pages
  1. Under Branch, change None to main, then click Save

You'll see "GitHub Pages source saved."

Wait about 90 seconds, then refresh the page.

Step 4 — View Your Live Site

You'll now see a message like:

Your site is live at:

https://username.github.io/firstrepo/

Click Visit site — this is your GitHub Page.

Step 5 — Understand the Difference

GitHub Page (Your Website)

https://username.github.io/firstrepo/

This is your live website that anyone can visit.

GitHub Repository (Your Code)

https://github.com/username/firstrepo

This is where your code lives — like a folder for your project.

For Your Final Project

Submit the GitHub Page link (github.io), not the repository link (github.com).

Don't Have an Account? Create One First

Sign up for free in 2 minutes

Sign Up for GitHub

  1. Go to github.com
  2. Click the "Sign up" button (top right)
  3. Enter your email address
  4. Create a password
  5. Choose a username

Choosing a Username

Your username becomes part of your website URL (username.github.io), so choose wisely!

  • Good: johndoe, jane-smith, jsmith2026
  • Avoid: xXx_gamer_xXx, temp123456
  1. Complete the verification puzzle (proves you're human)
  2. Check your email and click the verification link
  3. Answer the personalization questions (or skip them)
  4. Select the Free plan

Account Created!

You now have a GitHub account. Go back to the Quick Start guide above!

Making Changes to Your Website

How to update your site after it's live

Edit Directly on GitHub

You can edit your website directly on GitHub — no downloads needed!

  1. Go to your repository: github.com/username/firstrepo
  2. Click on index.html
  3. Click the pencil icon (top right of the file view) to edit
  4. Make your changes
  5. Click "Commit changes..."
  6. Click "Commit changes" again to confirm
  7. Wait about 90 seconds, then refresh your website to see the changes!

Pro Tip

As you advance, you'll learn to use a code editor on your computer and Git (the command-line tool) for a smoother workflow. For now, editing directly on GitHub works great!

Troubleshooting

Common problems and solutions

Pro Tip: Use AI for Help

Stuck? Describe your problem to your LLM of choice (ChatGPT, Claude, Gemini). You can even share screenshots — AI can read images!

Common Issues

"404 - Page Not Found"

  • Wait about 90 seconds — GitHub Pages needs time to build your site
  • Make sure your file is named exactly index.html (not Index.html or index.HTML)
  • Check that GitHub Pages is enabled in Settings → Pages
  • Make sure the Branch is set to main (not "None")

"I can't find the Settings or Pages option"

Make sure you're on your repository page (not your profile page). The URL should look like: github.com/username/firstrepo

"My changes aren't showing up"

  • Wait about 90 seconds after committing
  • Hard refresh your browser: Cmd+Shift+R (Mac) or Ctrl+Shift+R (Windows)
  • Try opening in an incognito/private window

"I don't see the 'creating a new file' link"

If your repository already has files, click the Add file button instead, then select Create new file.
